Data Engineer làm gì?

Xây hạ tầng để data chảy đáng tin cậy, nhanh, tiết kiệm — từ nguồn đến người dùng cuối.

02 · 4 trụ cột công việc

Ingest → Transform → Store → Serve.

01
Ingestion
Đưa data vào hệ thống. Batch (DB/file/API), streaming (click/IoT/log), CDC từ OLTP.
02
Transformation
Clean null/dup/outlier · Enrich join context · Aggregate theo ngày/tuần · Reshape wide↔long.
03
Storage
Data Lake (S3) · Warehouse (Redshift) · Operational (DynamoDB, Aurora). Chọn đúng theo pattern truy cập.
04
Serving
BI dashboard · DS notebook · Embedded analytics · ML feature store. Ai dùng quyết định shape data.
DE = foundation. Analyst và Data Scientist phụ thuộc hoàn toàn vào chất lượng pipeline bạn build.

02 · Cuộc dịch chuyển lớn

ETL → ELT. Vì sao?

Truyền thống
ETL · Extract → Transform → Load
Transform chạy trên ETL server riêng trước khi vào warehouse. Raw data không được giữ lại — muốn fix bug phải đọc lại từ source.
Cloud-native
ELT · Extract → Load → Transform
Load raw vào S3/warehouse trước, transform sau bằng SQL. Raw data immutable, có thể reprocess cho use case mới.
Vì sao ELT thắng?
Storage rẻ (S3 ~$0.023/GB) · Compute on-demand (Athena, Redshift Serverless) · Giữ raw data cho use case tương lai — bạn không biết trước data này sẽ cần cho cái gì trong 2 năm tới.

03 · Medallion architecture

Data Lake Zones — Bronze · Silver · Gold.

Bronze · Raw
Landing zone
Data y nguyên từ nguồn, immutable. Format: JSON, CSV, log gốc. Never delete, never modify.
Silver · Curated
Cleaned & validated
Đã clean null/dup, validated schema, deduplicated. Format: Parquet, Delta.
Gold · Consumption
Business-ready
Aggregated theo domain. Parquet partitioned, optimized cho BI + ML.
Quy tắc vàng
Raw không bao giờ bị xóa hay sửa. Muốn fix bug → rerun pipeline từ raw. Đây là safety net cho toàn bộ data platform.

03 · Batch vs Streaming

Khi nào dùng gì?

Câu hỏi này quyết định 50% kiến trúc của bạn — và 80% chi phí.

Tiêu chíBatchStreaming
LatencyPhút → giờ → ngàyMili-giây → giây
ThroughputRất cao (TB/job)Cao nhưng đắt hơn
ComplexityThấpCao — state, watermark, late data
CostRẻĐắt 3–10×
Use caseBáo cáo ngày, ML trainingFraud detection, realtime dashboard, IoT
Lời khuyên vàng: Luôn bắt đầu bằng batch. Chỉ chuyển streaming khi business thực sự cần sub-minute latency.

03 · Metadata · Governance · Security

Platform ≠ Pipeline.

Pipeline là code. Platform là code + catalog + governance + security. Thiếu 3 cái sau, bạn có pipeline nhưng không có platform.

Metadata & Catalog
Tìm data như tìm Google
  • Schema registry — tránh breaking change giữa producer/consumer
  • Data catalog — tìm table nào có field gì
  • Lineage — data này từ đâu, chảy đi đâu
  • AWS: Glue Catalog + Lake Formation làm cả 3
Governance & Security
Đủ chặt cho compliance
  • Access control — row/column-level (Lake Formation)
  • PII detection — Macie scan tự động
  • Encryption — at rest (KMS) + in transit (TLS)
  • Audit — CloudTrail + data access logs

04 · Decision tree

Nhanh — đây là cái bạn cần.

? Cần query SQL trên data trong S3?
  ├─ Ad-hoc, không lặp lại       →  Athena
  └─ Chạy thường xuyên, join nhiều →  Redshift Spectrum / Redshift

? Cần ETL?
  ├─ Serverless, nhẹ             →  Glue
  ├─ Spark jobs lớn              →  EMR / EMR Serverless
  └─ Event-driven, nhỏ           →  Lambda

? Cần streaming?
  ├─ Cần replay, custom consumer →  Kinesis Data Streams
  ├─ Chỉ cần đẩy vào S3/Redshift →  Firehose
  └─ Đã có Kafka                 →  MSK

04 · Cost optimization

5 thói quen tiết kiệm $10k/tháng.

S3
Intelligent-Tiering
Lifecycle → Glacier cho cold data. Có thể giảm 50–80% cost storage.
Athena
Parquet + partition
Giảm 90% chi phí scan. Pattern quan trọng nhất trong data lake.
Redshift
Pause / Serverless
Pause cluster khi không dùng hoặc dùng Redshift Serverless.
EMR
Spot instances
Giảm 70–90% cho task nodes. Spot OK với Spark vì có checkpoint.
Glue
Job bookmark
Incremental processing — tránh reprocess cùng data mỗi lần chạy.
Mindset
Engineer biết tiết kiệm $10k/tháng cho công ty = engineer đắt giá.

05 · Học từ failure

5 sai lầm đốt tiền & đốt niềm tin.

Pitfall 01
Streaming-first cho mọi thứ
Đốt tiền vô lý. 80% use case batch là đủ. Chỉ streaming khi sub-minute latency thực sự là business requirement.
Pitfall 02
Không partition S3 data
Athena scan full, bill shock. Một query đáng $0.01 thành $100.
Pitfall 03
Skip data quality checks
Downstream dashboard sai → mất niềm tin. Một khi mất, rất khó lấy lại.
Pitfall 04
Monolithic pipeline
Fail 1 bước, rerun cả 4 giờ. Thiết kế idempotent + checkpoint ngay từ đầu.
Pitfall 05
Không có catalog
6 tháng sau không ai biết table nào là gì. Data lake biến thành data swamp. Catalog không phải "nice to have" — là "must have" từ ngày 1.
