The Challenge: Multi-Tenant Data Leaks

The Problem

When multiple tenants (companies or users) upload codebases and documents to a centralized RAG system, there is a high risk of "cross-tenant data leakage" where an LLM hallucinates or accidentally retrieves proprietary code belonging to another tenant.

The Requirement

Build a robust governance system that strictly enforces data isolation at the retrieval layer while maintaining high performance and cost-efficiency.

The Solution: GraphRAG & Multi-Agent MCP

Key Technologies

  • GraphRAG: Maps relationships between code files to understand context beyond simple vector similarity.
  • Model Context Protocol (MCP): Safely scans documents and enforces licensing rules before exposing them to the RAG pipeline.
  • Multi-Agent System: Employs specialized agents (Scanner Agent, Retrieval Agent, Supervisor Agent) to validate every step.

Reference AWS Pattern: Multi-tenant Knowledge Base (Pool Pattern)

AWS Services Architecture

How it translates to AWS

To replicate this winning pattern for the W7 Capstone (StudyBot/DocHub), you should utilize:

  • Amazon Bedrock Knowledge Bases - Set up a Pooled Pattern index.
  • Amazon OpenSearch Serverless - Vector storage with strict Metadata Filtering (e.g., filtering by tenant_id).
  • Amazon Bedrock Agents - Create an orchestration agent to route queries based on tenant permissions.
  • AWS Lambda - Act as Action Groups for MCP implementation and licensing validation.
