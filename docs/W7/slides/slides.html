📚 EduTech: Agentic RAG

Project: Ivy (AWS 10,000 AIdeas 2026)

Real-World Solution: Moves beyond basic RAG to Self-Corrective RAG. The system autonomously plans learning paths based on error histories and handles low connectivity gracefully.

💰 FinTech: Hybrid Multi-Agent

Project: Sentinel (GenAI Case 2025)

Real-World Solution: Avoids LLM-only approaches to cut costs. Uses a Hybrid Workflow: Regex for easy transactions, Supervisor Agent for complex ones. Adds Human-in-the-loop if confidence is low.

🏢 Doc Hub: GraphRAG & MCP

Project: Privacy Guardian (GenAI Impetus 2025)

Real-World Solution: Solves cross-tenant data leaks via Model Context Protocol (MCP) and GraphRAG. A multi-agent network safely scans documents before exposing them to the RAG pipeline.

💡 Demo Pro-Tip: Don't just say "We used Bedrock". Say "We combined rule-based filtering with Bedrock Agents to optimize cost and introduced a review queue for low-confidence outputs."

Data Flow (Pool Pattern)

Source: AWS Machine Learning Blog (Advanced Knowledge Bases Architecture)

1. Cost Discipline

No separate DBs per tenant. Uses a single OpenSearch Pool separated only by metadata tags.

2. Enterprise Security

Forces tenant_id injection at the Lambda Authorizer level. Prevents frontend tampering or spoofing.

3. Anti-Hallucination

Queries DynamoDB to filter out stale documents before RAG, ensuring the AI only cites the latest versions.

7 Mandatory Capabilities

Every group MUST demonstrate all of these on Friday. Service choices must be justified.

1 User-Facing Entry: Public HTTPS (e.g., CloudFront, ALB, API GW)

2 Application Compute: Where code runs (e.g., Lambda, EC2, ECS)

3 AI / ML Feature: Bedrock KB+Agent, or InvokeModel

4 Data Persistence: User state across sessions (e.g., DynamoDB, RDS)

5 Object Storage: Files/Blobs (S3)

6 Network Foundation: DB not public-facing (VPC, private subnets, SGs)

7 Identity & Access: IAM least-privilege roles for compute. (User auth is your choice: IAM/Cognito/Custom)

"Why did you pick this service for this capability?"
A strong answer connects to trade-offs (cost, speed, scale). "Because we knew it" is a weak answer.

Cost Discipline: $100 Hard Cap

Your real account, your real money. Exceeding $100 caps your deployment score to a maximum of 3.

Cost Survival Tips:
  • Use cheapest sufficient model (e.g., Claude 3 Haiku) during dev.
  • Avoid Multi-AZ RDS. Use single-AZ db.t3.micro.
  • Skip NAT Gateway ($1.50/day) if possible; use VPC Endpoints.
  • Delete unused resources daily (e.g., idle EC2s).
