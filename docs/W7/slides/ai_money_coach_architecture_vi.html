Vì sao không chọn RAG / Agent?

Dùng đúng công cụ cho đúng loại dữ liệu.

Direct Bedrock

Transaction row đã có sẵn trong request. Model chỉ cần trả category và confidence có cấu trúc.

Knowledge Base / RAG

Hợp với hỏi đáp tài liệu. Với classification v1, retrieval thêm ingestion, vector store và cold-start không cần thiết.

Bedrock Agent

Return of Control dùng cho action orchestration. Review queue hiện là business workflow đơn giản trong ứng dụng.

Database decision

Spending insight là bài toán SQL aggregation.

Không chọn DynamoDB cho v1

DynamoDB thắng khi truy cập theo key ở throughput cao. Với summary theo thời gian và category, hệ thống cần thêm bảng tổng hợp, GSI/pipeline hoặc scan không phù hợp cho demo hiện tại.

RDS Single-AZ cho hackathon; RDS Proxy giảm rủi ro connection burst từ Lambda.

Security & observability

Insight tài chính chỉ có giá trị khi truy vết được.

Network + IAM

  • RDS private, inbound chỉ từ Lambda security group.
  • S3 Block Public Access và encryption.
  • DB credential ở Secrets Manager.
  • Lambda role chỉ cấp quyền bucket/model/secret cần dùng.

Full Observability

  • TransactionsClassified
  • RuleMatchRate
  • NeedsReviewRate
  • BedrockLatencyMs + alarm lỗi API
