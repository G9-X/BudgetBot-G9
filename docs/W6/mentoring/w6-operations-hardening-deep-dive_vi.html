Cost cap

$150 không phải “budget tham khảo”. Nó là hard gate.

Fail

Nếu account total > $150, W6 fail dù 4 MH nhìn có vẻ đẹp.

Default choices

  • Single-AZ cho workshop/dev
  • Smallest instance đủ demo
  • Serverless khi hợp lý
  • Tắt compute qua đêm

Forbidden smell

  • Bedrock Provisioned Throughput
  • OpenSearch multi-node
  • EKS cho demo ngắn
  • RDS Multi-AZ “cho đẹp”

Ở W6, “biết chọn tài nguyên rẻ hơn và chứng minh đã kiểm soát spend” chính là năng lực cloud engineering.

Research-backed trap

AWS yêu cầu user-defined cost allocation tags phải được activate riêng trong Billing console trước khi xuất hiện trong Cost Explorer / cost reports. Tag có thể mất tới 24h để xuất hiện.

Nguồn research: AWS Billing & Cost Management docs về cost allocation tags.

MH-COST-A pattern

Automated Cost Guard: observe → decide → stop → prove

# Lambda sketch: stop compute without keep=true
def handler(event, ctx):
  ec2 = boto3.client('ec2')
  for i in list_running_instances(ec2):
    tags = {t['Key']: t['Value'] for t in i.get('Tags', [])}
    if tags.get('keep').lower() != 'true':
      ec2.stop_instances(InstanceIds=[i['InstanceId']])
      log('stopped', i['InstanceId'])

Two triggers

Scheduled trigger: EventBridge cron daily/overnight — reliable in 48h account.

Cost path: Budgets $150 → SNS → Lambda — wire and demo with test SNS publish.

Latency ADR

Cost data lags ~8–24h. Trong workshop, real budget event có thể chưa fire. Điều này OK nếu schedule path đã stop thật và ADR giải thích production behavior.

Least privilege

Role của Cost Guard không được là “* vì cho nhanh”.

Không đạt

{
 "Action": "*",
 "Resource": "*"
}

Trainer hỏi “vì sao least privilege?” là sập.

Đạt hơn

{
 "Action": [
  "ec2:DescribeInstances",
  "ec2:StopInstances",
  "rds:DescribeDBInstances",
  "rds:StopDBInstance"
 ],
 "Resource": "*"
}

Describe thường cần Resource *, nhưng action set phải hẹp.

CloudWatch Agent

Memory/disk không tự xuất hiện trong CloudWatch.

Default EC2 metrics không có memory/disk utilization. Muốn monitor tài nguyên OS-level, phải cài CloudWatch Agent và publish sang namespace như CWAgent.

Alarm trên metric chưa có datapoint sẽ ở INSUFFICIENT_DATA. Nếu chờ tới chiều thứ Năm mới cài agent, rất dễ không có lịch sử metric cho Friday demo.

Walkabout checklist

  • CWAgent namespace có datapoint.
  • Dashboard widget không rỗng.
  • Ít nhất một alarm OK hoặc ALARM.
  • Alarm có action destination.
Log Insights

Query tốt phải tạo insight, không chỉ list log.

Lambda error spikes

fields @timestamp, @message
| filter @message like /ERROR/
| stats count(*) as error_count by bin(5m)
| sort @timestamp desc

AccessDenied by principal

filter errorCode="AccessDenied"
| stats count(*) by userIdentity.arn
| sort count desc

Yêu cầu evidence: query text, log group thật, kết quả có rows, và saved query name.

MH-SEC pattern

Self-Healing Security Guard = cùng pattern với Cost Guard, nhưng sửa misconfig.

Near real-time
CloudTrail API event→EventBridge rule→Lambda→Revoke SG / Put BPA
Fallback reliable
EventBridge Scheduler→Scan current state→Fix violation→CloudTrail evidence

Clean target A

Security Group open to 0.0.0.0/0 on SSH/RDP → Lambda calls RevokeSecurityGroupIngress.

Clean target B

S3 bucket public → Lambda calls PutPublicAccessBlock to re-enable bucket-level Block Public Access.

Security Guard code

Demo phải chứng minh “fix API call” thật sự chạy.

# SG auto-remediation sketch
def handler(event, ctx):
  ec2 = boto3.client('ec2')
  for sg in risky_security_groups(ec2):
    for perm in sg['IpPermissions']:
      if open_to_world_ssh_or_rdp(perm):
        ec2.revoke_security_group_ingress(
          GroupId=sg['GroupId'], IpPermissions=[perm])

Evidence không thể thiếu

  • Before: violation tồn tại.
  • After: rule/bucket đã được fix.
  • CloudTrail event: RevokeSecurityGroupIngress hoặc PutPublicAccessBlock.
  • Lambda execution role least-privilege.
Preventive control

Self-healing cần đi kèm một guardrail phòng ngừa.

PathKhi chọnEvidence cần showCost statement
KMS CMKApp có data store cần audit decrypt/useCMK rotation, resource encrypted by CMK, CloudTrail GenerateDataKey/DecryptCMK ~$1/month/key; justified by audit trail
S3 BPA + deny policyApp có S3 data/corpus bucketAccount-level BPA ON, bucket policy deny non-TLS/non-encrypted, denied testNear-zero cost; reduces public data blast radius
IAM Access AnalyzerCần triage external accessFinding reviewed + resolved/justifiedOperational review cost vs exposure risk
KMS mental model

KMS không “giữ data của bạn”. Nó quản lý key hierarchy.

1

Generate data key

Service/app gọi KMS để lấy plaintext data key + encrypted data key.

2

Encrypt locally

Data được encrypt bằng data key; KMS không thấy nội dung plaintext application data.

3

Audit decrypt

Khi cần decrypt, encrypted data key được gửi tới KMS; CloudTrail cho biết principal/service nào dùng key.

Nguồn research: AWS KMS developer guide về data keys và calls qua KMS.

Rubric lens

Strong pass

Exceeds

Close
